# 小型图书借阅管理系统

## 一、项目简介
本项目是基于Python命令行的小型图书借阅管理系统，为《计算机工程训练》实验四单人开发项目。实现图书录入、借阅、归还、查询等核心功能，采用模块化设计，结合AI辅助编码完成。
## 二、核心功能
### 必做核心功能
1. 图书信息录入：支持录入图书编号、书名、作者、库存，自动校验编号唯一性
2. 图书借阅：输入图书编号和借阅人姓名，扣减库存并记录借阅人
3. 图书归还：输入图书编号和归还人姓名，恢复库存并清除借阅记录
4. 模糊查询：支持按书名/作者关键词查询图书，不区分大小写
5. 图书列表展示：查看所有图书的库存、借阅状态、借阅人信息

### 可选扩展功能
1. JSON 数据持久化：数据保存到本地文件，程序重启不丢失
2. 按库存排序：图书列表支持按库存数量从高到低排序
## 三、运行方法

2. 克隆本仓库到本地
3. 终端进入项目目录，执行启动命令：

## 四、模块设计
本项目采用三层模块化架构，符合高内聚低耦合设计原则，各模块职责清晰，通过统一接口交互。

### 整体文件结构
项目根目录 /
├── book.py # 实体层：图书数据模型
├── library_manager.py # 业务层：核心业务逻辑
├── main.py # 交互层：命令行用户界面
├── README.md
└── .gitignore
### 各模块职责
1. **实体层（book.py）**：封装图书数据模型，定义图书属性与序列化方法，不包含业务逻辑
2. **业务层（library_manager.py）**：核心业务逻辑处理，管理图书数据与借阅规则，是系统核心
3. **交互层（main.py）**：用户命令行交互，展示菜单、接收输入、调用业务接口，不包含业务逻辑
### 接口约定
业务层所有方法统一返回 `(bool, 提示信息/数据)` 二元组：
- 成功：`(True, 成功提示/结果数据)`
- 失败：`(False, 错误原因字符串)`

| 方法名 | 所属模块 | 功能说明 | 输入参数 | 返回值 |
|---|---|---|---|---|
| add_book | library_manager.py | 添加新图书 | book_id, title, author, stock | (bool, 提示信息) |
| borrow_book | library_manager.py | 借阅图书 | book_id, borrower_name | (bool, 提示信息) |
| return_book | library_manager.py | 归还图书 | book_id, borrower_name | (bool, 提示信息) |
| search_book | library_manager.py | 模糊查询图书 | keyword | (bool, 匹配的图书列表) |
| get_all_books | library_manager.py | 获取全部图书 | 无 | (bool, 所有图书列表) |

### 业务规则
- 图书编号全局唯一，重复编号禁止添加
- 库存必须为非负整数，借阅后库存不可为负
- 归还时必须校验借阅人是否在借阅记录中
- 模糊查询不区分大小写，同时匹配书名和作者
## 五、AI 使用说明
### 1. AI 使用场景
- 借助 AI 生成 Book 类的基础框架与序列化方法，减少重复编码
- 使用 AI 辅助编写 LibraryManager 类的核心业务逻辑，提供功能实现思路
- 遇到输入校验、异常处理等边界问题时，向 AI 咨询解决方案
- 代码出现 bug 时，将报错信息发给 AI，辅助定位问题原因

### 2. AI 带来的帮助
- 快速搭建项目基础结构，缩短编码时间，将更多精力放在业务逻辑设计上
- 提供多种异常处理思路，补充了未考虑到的边界情况（如负库存校验、重复编号校验）
- 优化交互菜单的输入校验逻辑，提升程序的健壮性
- 提供代码规范建议，让变量命名、注释格式更统一

### 3. 遇到的问题与修正方法
- **问题1**：AI 初始生成的模糊查询方法区分大小写，输入小写关键词搜不到大写开头的书名
  - 修正方法：手动修改代码，将关键词和书名、作者都转为小写再比较，实现不区分大小写的模糊匹配

- **问题2**：AI 生成的借阅功能没有校验空输入，直接传空字符串会导致逻辑错误
  - 修正方法：在交互层添加空输入校验，所有必填项为空时提示用户重新输入

- **问题3**：默认参数 `borrowers=[]` 会导致所有实例共享同一个列表
  - 修正方法：将默认参数设为 `None`，在 `__init__` 方法里判断为空时再创建空列表

### 4. AI 使用反思
AI 是很好的辅助编码工具，能快速生成基础代码，提高开发效率，但 AI 生成的代码不能直接使用，必须经过人工审查和测试。AI 经常会遗漏边界情况，而且对业务需求的理解可能有偏差，核心逻辑必须自己把控，确保每一行代码的逻辑都能理解和解释。
